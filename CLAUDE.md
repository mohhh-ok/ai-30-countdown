# プロジェクト方針メモ

## 作業を始める前に（最初に読む）
- **ユーザーの意図を正しく理解するため、着手前にまずドキュメントへ目を通すこと。**
  - `README.md`: 実装の全体像・LLM バックエンド切替・仕組み。
  - `docs/`: 個別トピックの知見（`image-gen.md` / `runpod-serverless.md` など）。
  - この `CLAUDE.md`: Git 運用・LLM 呼び出し・画像生成の方針。
- 指示が曖昧なときも、これらの方針に照らして意図を汲んでから動くこと。
- なお `plan.md` は初期の v1 仕様で**内容が古い**ため、原則として参照しない。

## Git 運用
- これは個人プロジェクト。他インスタンス（別の Claude/作業）との作業分離は気にしなくてよい。
- 作業ブランチを切って隔離する必要はなく、**`main` に直接コミット＆push してよい**。
- 「commit/push して」と言われたら、ブランチを作らず `main` でそのまま進めてよい。

## LLM 呼び出し方針
- **本番（公開運用）では `claude -p`（Claude Code CLI）を使わない。**
  - Claude Code / サブスクは「個人が対話的に開発する」用途が前提。シミュレーションを自動で回し続け、その出力を YouTube やサイトで公開する＝「Claude を組み込んだサービスの運営」に近く、サブスク経由 CLI の想定用途から外れる。
  - 公開フェーズの本番ループは **Anthropic API（従量課金・商用ターム）クライアント**に寄せる。レート制限・ポリシーの両面で安全。
- ローカルで自分が見て楽しむ／身内に見せる程度の開発・検証段階に限り、`claude -p` 運用でよい。
- 設計上、LLM 呼び出し層は **CLI / API を差し替えられる構造**にしておく。

## 画像生成方針
- **キャラ絵などの画像生成は OpenAI `gpt-image-2` を使う。**（`OPENAI_API_KEY` / 従量課金 API）
- ただし**背景透過が欲しいときは `gpt-image-1`**（gpt-image-2 は透過非対応）。
- 出力は `cwebp` で WebP に変換して `assets/` に保存（軽量化＋透過保持）。
- 生成スクリプト・モデルの使い分け・知見は `docs/image-gen.md` を参照。
