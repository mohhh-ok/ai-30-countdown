# Research Logs (research)

A place for experiment records that observe real LLM behavior "in numbers, under fixed conditions."

## decide-samples.jsonl — Observing the distribution of action choices (issue #7)

The action choices of `claude -p` are unpredictable. This is a record for **throwing the same world state at the real LLM N times and seeing, as a distribution, which action each character leans toward** (1 experiment = 1 JSONL line, append-only — never deleted).

- Generated by: `scripts/sample-decide.ts`
- mock (`src/llm/mock.ts`) is deterministic, so it is out of scope. Run **the real LLM only**.

### Example run

```bash
bun scripts/sample-decide.ts \
  --purpose "利他70・同室ありで share が出るか" \
  -n 10 \
  --roster haru,nagi \
  --set haru.energy=30 haru.satiety=40 haru.params.altruism=70 \
  --set haru.place=kibune nagi.place=kibune
```

- `--purpose` (required): what the experiment is meant to confirm.
- `-n` (default 10): how many times the same state is thrown at the real LLM.
- `--roster` (default: all characters): character ids to cast (comma-separated).
- `--set <id>.<path>=<value>`: overrides for the world state. `<path>` is `energy` / `satiety` /
  `stealBurden` / `lonelinessSensitivity` / `params.altruism` etc., or `place` (= place id).
  Unknown ids, unknown fields, and non-numeric values throw and stop (no silent defaulting).
- `--mode parallel|combined` (default: depends on the backend) / `--weather normal|lean`.

### Fields of one record

`datetime` (+09:00) / `commit` (HEAD short) / `backend` / `mode` / `purpose` / `n` /
`weather` / `condition` (the main parameters and places that were fixed) / `rooms` (who shares a room) /
`results` (action → count) / `errors` (failure count) / `rawSamples` (the raw sample sequence).

LLM response parse failures and invalid actions are not swallowed — they are recorded and displayed as **ERROR** and excluded from the distribution.

### Notes

- This hits the real LLM N × (number of characters) times, so it consumes time and quota.
- When using `claude -p`, do not put `ANTHROPIC_API_KEY` in the environment (run on OAuth subscription auth; see CLAUDE.md).
